# Tailscale 当前设备列表设计

## 目标

在“对端”页面的现有用户/设备两层列表中展示当前 OpenWrt 设备。当前设备应归入其 Tailscale 用户分组，并与其他设备共享状态、地址、角色、已发布网段、筛选和分页逻辑。

## 数据处理

`tailscale status --json` 将当前设备放在顶层 `Self`，其他设备放在 `Peer`。页面解析状态时应使用同一套规范化逻辑处理两类节点，并为 `Self` 增加 `isSelf: true` 标记。

当前设备使用 `Self.UserID` 查找顶层 `User` 中的显示名称和登录名，因此与该用户的其他设备显示在同一分组下。若 `Self` 缺失，则保持现有行为，仅显示 `Peer`。

## 页面行为

- 当前设备与普通设备使用相同的名称、Tailnet IP、在线状态、最后在线时间、角色和已发布网段列。
- 当前设备参与“全部”“在线”和“发布子网”筛选。
- 当前设备参与设备数量统计和现有分页。
- “探测”列对当前设备固定显示“当前设备”，不渲染探测按钮，也不允许调用探测命令。
- 普通在线、离线设备的现有探测行为保持不变。

## 测试

- 验证 `Self` 被规范化并带有 `isSelf` 标记。
- 验证 `Self` 根据 `UserID` 进入正确的用户分组。
- 验证缺少 `Self` 时保持兼容。
- 验证当前设备行显示“当前设备”且没有可点击的探测按钮。
- 运行现有对端页面测试和完整项目测试，确认筛选、分页、探测及打包检查无回归。
